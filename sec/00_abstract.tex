\begin{abstract}
Speculative decoding accelerates large language model inference by using a smaller draft model to propose token blocks that a target model verifies in parallel. Despite a substantial recent literature, reported speedups vary widely with hardware, model pairing, draft length, and sampling regime, and most published numbers come from data-center GPUs. We present a controlled empirical study on consumer hardware (NVIDIA RTX~5090, \texttt{int8} target via \texttt{bitsandbytes}) using Qwen2.5-7B-Instruct as the target and Qwen2.5-0.5B/1.5B as same-family drafts, evaluated on GSM8K, MMLU subsets, and CNN/DailyMail with frozen sample manifests. We measure throughput, time-to-first-token, time-per-output-token, token acceptance dynamics, and task quality across draft lengths $k\!\in\!\{4,8,16\}$ and two sampling regimes. The best autoregressive configuration delivers a wall-clock speedup of $1.65\times$ at $k{=}8$ ($\bar{\alpha}{=}0.40$, $\bar{B}_{\text{eff}}{=}3.20$), with quality changes bounded within $1.0$ point on GSM8K and MMLU. The empirical effective block size tracks the geometric upper bound $(1{-}\alpha^{k+1})/(1{-}\alpha)$ within $3\%$, isolating the linear-in-$k$ draft cost as the binding term once per-block overhead is amortised at $k{\geq}8$. Building on this analysis, we propose \emph{DriftDiffuse}, a $\sim$30\,M-parameter bidirectional masked-diffusion drafter with a position-conditional drift schedule that proposes $k$ tokens in $n_{\text{denoise}}{\ll}k$ parallel steps while remaining compatible with the standard verifier. We provide a fully reproducible protocol and operating guidelines for selecting draft model size and draft length under explicit quality constraints.
\end{abstract}
